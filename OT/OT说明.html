设计说明 · 使用指南

📱 手机 / 电脑端 🔐 IDEA 账号登录 📊 SAP 报表导出

1. 登录方式

🔗 登录入口

支持二维码 / 链接方式登录，手机端与电脑端均可使用。无需 IDEA 账号即可登录；审批人和报表查看需要 IDEA 账号。

👤 员工身份校验

填写员工号时系统自动判断状态：仅 Active 状态员工显示登录按钮，离职员工限制登录。

🔑 修改密码

首次登录需修改初始密码。点击"修改密码"超链接，输入原始密码和两次新密码，提交后跳转到登录界面。

🔄 忘记密码 · 自助重置

  • 点击"忘记密码"，输入员工号，预留邮箱会收到一串随机码
  • 跳转到修改密码界面，输入收到的随机码和新密码
  • 提交后跳转到 OT 申请界面

2. 员工上岗 / 转岗 / 离职管理

📋 初始化

根据提供的数据一次性上传到《人员信息维护表》中，需提供活动邮箱用于通知审批结果、修改密码等。系统提供二维码，供用户自行扫码输入个人邮箱。

🆕 新员工录入

填写《员工信息更新-新增新员工》表，数据库按生效日期自动新增。考虑固定加班计算：生效日在前则立即生效并按月新增固定加班数据条目；生效日在后则当日新增。

🔄 转岗

填写《员工信息更新-员工信息变更》表，每天零点系统自动判断：当 valid from ≤ 今天时自动触发，修改所有改动的个人数据。当为班组人员时，才触发修改固定加班时长。

🚪 离职

填写《员工信息更新-员工信息变更》表，每天零点自动触发，修改所有改动的个人数据，并将当年剩余固定加班归零。

👥 考勤员权限

  • 可查看《人员信息维护》表（密码、验证码除外）—— 仅查看权限
  • 可编辑和查看《员工信息更新》表 —— 统一编辑入口
  • 修改后即触发，考虑晚维护情形

3. 日历管理

📅 法定假期 & 排班表

《国家法定假期》《综合工时人员排班表》已自动维护 30 年。

🔧 年度维护

每年初维护特殊调休日、公司假日等，OT rate 会自动计算。

📆 固定加班表

综合工时班组固定加班表统一维护 20 年（如国家法定假日增加可重新维护）。

🔄 年度自动生成

每年 1 月 1 日自动根据固定加班表和人员清单生成下一年的《综合工时人员排班表》，建议提前审核。数据工厂更新时间为每年初1月1日，频率为每年一次，人员筛选时去除 Deactive 离职人员。

📊 每月审核《倒班工人固定加班时数（按人）》

  • 考勤员每月查看固定加班时数表，可手动调整加减项
  • 如人员信息有误，去员工信息更新修改，固定加班时数表会自动更新
  • 考勤员每人仅限自己数据权限，提供 12 个月份快捷视图（提交时间 = 当年，月份 = 1–12）

4. 审批矩阵管理

👔 审批人员配置

提供各部门的一级审批负责人（多选）、二级审批负责人、考勤员，以上人员需要 BASF 账号接收审批消息。

📝 审批流程

倒班班长 / 部分没有一级审批的部门，直接进行二级审批；其余记录进行一级审批。填写 OT 申请表中的"审批意见"和"评论"字段。

📋 月底批量审批

部门经理于月底进行批量审批，批量自动填写 OT 申请表中的"经理意见"字段。

5. 最上层设计规则

a所有申请单每条数据不得跨年（强制提交校验限制）
b标准工时 C 和 F 不得跨月（强制提交校验限制）
c申请日最早提前 30 天，补登一周之内，超过不可补登
dOT 最小单位以半小时计
e抢修不纳入任何上限管理，报表区分总 OT 和纳入计算 OT
fCDEF 允许选择补休；B 选择补休后自动校验限制提交。默认结算工资。补休不计入成本但计入上限
g不同申请单不允许同一个人相同日期提交；一级审批驳回后清除校验
h同一申请单允许相同日期不同行提交（上下午两段），时间段重复校验
i培训加班 ≥ 8 小时，后台自动产生培训员工倒班津贴扣款 / 白班顶班津贴 -50
j培训类统一合并为"培训"；抢修自动分类：Rush Repair-PH / OFF / WORK
k一级审批进入工作流，多人任意一人批准即可
l二级审批月底批量审批，不进入工作流，结果回写主数据
mCost 字段仅开放 GM 和 Production Head（Admin）
nSAP 报表结构 follow 马来西亚格式，开启导出功能
o倒班 08:00–20:00，默认 0 吃饭时间，可选 0.5 hour
p常日班 08:30–17:00，默认 0.5h 吃饭时间，可选 0 hour
q24:00 PM 为非法字符，导出时自动转换为次日 00:00 AM

6. 分工时类别设计规则

E 类：综合工时 · 倒班人员（含 4 级倒班班长）

⏰ 工作时间控制

只能填写非工作时间，系统根据排班和班组自动筛选可选时间。上班后至少休息8小时。考虑班长特殊培训情况。

🍽 吃饭时间

默认 0 hr，提供手动可选择项 0.5 hr。如选择，后台直接记录 12:00–12:30 PM。

💰 OT Rate

国家假期 ×3.0，其他 ×1.5。

📊 上限管理

每年加班 432 小时提交限制。固定加班总时长纳入上限管理，不上报马来西亚。申请页面显示本次总申请小时数和纳入上限计算总小时数。

🧮 固定加班计算逻辑

  • 各班组每月工作总时长 − 常日班每月工作总时长（165.36）
  • 计算多年每个班组总时长的总表，分到月份
  • 每年自动生成匹配到每个人的表，考勤员每月确认可手动修改
  • 根据人员变更触发智能助手修改后续月份数字

⚠️ 连续加班 ≤ 12 小时

  • 倒班工作后加班必须休息8小时，控制小时选择范围。
  • 例外：班长 sleep 班后早上开会到 12 点（OT 类别为培训会议时，额外开放 08:00–12:00，保留16:00 后开放）。
  • 豁免：抢修情形不受此限制。
C 类：标准工时 · 特殊 4 级员工（MAI、MDI 生产白班）  |  F 类：其他人员

⏰ 工作时间

常日班 08:30–17:00。工作日或调休工作日加班时间系统强制控制；假期和调休假日不控制选择时间。

🍽 吃饭时间

默认 0.5 小时，提供 0 小时选项。如不为 0，后台自动填写 12:00–12:30 用于 SAP 数据上报。

💰 OT Rate

法定假日 ×3.0，周末和调休周末 ×2.0，其他 ×1.5。

📊 上限管理

每月加班 ≤ 36 小时（强制提交限制）；工作日加班 ≤ 3 小时/天（含多条一天的情况）；周末/调休假日/国家假日 ≤ 11 小时/天。

🛌 每周至少休息一天

周度校验：总休息日 − 非工作日提交计数 ≥ 1，否则警告并强制填写补充说明。

B 类：不定时 · 除 C/D/E 类外的所有 4 级员工（工程师、副经理）

📅 申请限制

只有法定假日可申请加班，后台强制校验，不满足限制提交。

🍽 吃饭时间

默认 0.5 小时，提供 0 小时选项。如不为 0，后台自动填写 12:00–12:30 用于 SAP 数据上报。

💰 OT Rate

统一 ×3.0。

⏱ 单日上限

单日加班不可超过 11 小时，限制提交。

D 类：综合工时 · 特殊 4 级员工（行政主管、IT 主管、LFG 主管）

📅 法定假日

法定假日的加班必须为结算工资，限制提交。

🔄 其他日期

其他日期申请加班计入上限，类型是补休，限制提交。

📊 上限管理

每年加班不超过 432 小时，限制提交。
